Verify bank details

Prototypes

Error messages if verify fails
Use to collect data in the same way as before, showing error messages when the backend service is not able to verify the account details. Do not stop the user from the completing the application — this would need to be manually processed with a follow up contact to collect/verify the details are correct.

Get different details depending on the bank
Use this to only ask for the data we know we need based on the holding bank rather than asking for all the information with some optional fields. This would require data of the rule sets for each banks payment service.
We have not seen/validated a user need for this.

Pattern principals

  • bank account details are verified via a third party
  • don't soft credit check people, we just need to know if an account number is part of the sort codes data range
  • don't ask for information we don't need
  • give people confidence their transaction in secure
  • allow people to complete the transaction even if they don't have a bank account
